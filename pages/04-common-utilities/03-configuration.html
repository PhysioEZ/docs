Configuration & Utilities

The common/ directory contains files for managing system-wide settings and providing shared helper functions. This guide covers config.php for constants and secrets, and utils.php for reusable utility functions.

Configuration (`config.php`)

The config.php file is the central repository for all system-wide constants, credentials, and secret keys. This file should be included wherever these constants are needed, but it **must never be committed to version control with production secrets**.

It is typically used to define:

  • Database connection credentials.
  • Encryption keys for sensitive data (e.g., chat messages).
  • API keys for third-party services.

Example `config.php`

<?php
// common/config.php

// --- Database Configuration ---
define('DB_HOST', 'localhost');
define('DB_USER', 'root');
define('DB_PASS', '');
define('DB_NAME', 'caresync_os');

// --- Encryption Key for Chat ---
// This key is used to encrypt and decrypt chat messages.
define('CHAT_ENCRYPTION_KEY', '2L92k78hExeiUiS1xQTBP8VQciGyLcAQkNPNWilGgC0');

?>
Security Warning: The config.php file should be added to your .gitignore file to prevent accidentally exposing sensitive credentials in your repository.

Utilities (`utils.php`)

The utils.php file is a collection of general-purpose helper functions that can be used anywhere in the application. According to the architecture, this file is intended for shared logic like date/time formatting and other common tasks.

By centralizing these functions, you avoid code duplication and ensure consistent data formatting across all modules.

Example Usage

Imagine you have a function in utils.php to format dates. You would use it like this: 

<?php
require_once '../../common/utils.php'; // Include the utilities file

$database_timestamp = '2025-11-21 14:30:00';

// Use a hypothetical helper function from utils.php
$friendly_date = format_date_for_display($database_timestamp);

echo $friendly_date; // e.g., "November 21, 2025"
?>
